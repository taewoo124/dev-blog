# 정의

---

쿠키는 웹 서버가 생성하여 웹 브라우저로 전송하는 작은 정보 파일이다. 웹 브라우저는 수신한 쿠키를 미리 정해진 기간 동안 또는 웹 사이트에서의 사용자 세션 기간 동안 저장한다. 웹 브라우저는 향후 사용자가 웹 서버에 요청할 때 관련 쿠키를 첨부한다.

쿠키는 웹 사이트에 사용자에 대한 정보를 제공하여 웹 사이트가 사용자 경험을 맞춤화하는 데 도움이 된다.

인터넷에서 사용되는 쿠키를 **_HTTP 쿠키_** 라고도 한다. 대부분 웹과 마찬가지로 쿠키는 HTTP 프로토콜을 사용하여 전송된다.

---

# 구조

---

쿠키는 이름-값 쌍으로 구성되며, 선택적으로 추가 속성을 가질 수 있다.

쿠키의 각 부분은 세미콜론(;)으로 구분된다.

```lua
name=value; expires=date; path=path; domain=domain; secure
```

`name=value`

- 쿠키의 기본 부분으로, 쿠키에 저장된 데이터를 나타낸다.
- “name”은 쿠키의 이름을 나타내며, “value”는 해당 쿠키에 저장된 값을 나타낸다.

`expires=date`

- 쿠키의 만료 날짜를 설정한다.
- 이 속성이 설정되지 않으면, 쿠키는 브라우저 세션이 종료될 때 자동으로 삭제된다.

`path=path`

- 쿠키를 사용할 수 있는 URL경로를 설정한다.
- 이 속성을 통해 특정 페이지나 디렉토리에 대한 쿠키 접근을 제한할 수 있다.

`domain=domain`

- 쿠키를 사용할 수 있는 도메인을 설정한다.
- 이 속성을 통해 쿠키의 접근 범위를 제한하거나 확장할 수 있다.

`secure`

- 이 속성이 설정되면 쿠키는 HTTPS를 통한 연결에서만 전송된다.

# 어디에 저장될까?

---

쿠키는 사용자의 컴퓨터나 모바일 장치에 있는 웹 브라우저 디렉토리 또는 폴더에 저장된다. 쿠키는 사이트에 대한 특정 정보(로그인 상태, 장바구니, 항목 등)를 저장하고 해당 사이트를 재방문할 때 이 정보를 다시 가져와 사용한다.

쿠키의 위치는 사용하는 웹 브라우저와 운영 체제에 따라 다르게 저장된다.

Google Chrome에서는 모든 쿠키를 “Cookies”라는 파일에 저장한다.

개발자 도구에서 "애플리케이션" 탭을 클릭한 다음 왼쪽 메뉴에서 "쿠키"를 클릭하여 브라우저에 저장된 쿠키를 확인할 수 있다.

# 사용 용도

---

### 사용자 세션

---

쿠키는 사용자가 웹 사이트에서 수행하는 다양한 작업을 추적하는 데 사용된다.

사용자가 로그인을 하면 쿠키는 사용자의 로그인 정보를 저장하고, 사용자가 사이트를 이동할 때마다 그들이 로그인 상태를 유지할 수 있도록 한다. 이는 쇼핑 카트에 상품을 추가하는 경우에도 마찬가지로, 사용자가 웹사이트를 탐색하는 동안 선택한 항목을 기억하는 데 사용된다.

### 개인화

---

쿠키는 웹사이트의 사용자 경험을 개인화하는 데 사용될 수 있다.

웹 사이트는 사용자의 선호 언어를 쿠키에 저장하여, 사용자가 사이트를 재방문할 때마다 해당 언어로 사이트를 표시할 수 있다.

또는 웹 사이트는 사용자가 이전에 본 페이지나 검색어 등을 쿠키에 저장하여, 사용자에게 관련된 콘텐츠나 광고를 보여줄 수 있다.

### 추적 및 분석

---

쿠키는 사용자의 웹사이트 이용 패턴을 추적하고 분석하는 데도 사용된다.

웹사이트 소유자가 어떤 페이지가 가장 인기가 있는지, 사용자가 웹사이트에서 얼마나 오래 머무는지 등을 알아보는 데 도움이 된다. 이 정보는 웹사이트를 개선하고, 사용자에게 더욱 관련성 높은 콘텐츠를 제공하는 데 사용될 수 있다.

### 광고 타겟팅

---

쿠키는 사용자의 웹 사이트 방문 기록, 클릭한 광고, 검색어 등을 기반으로 사용자에게 맞춤화된 광고를 제공하는 데 사용될 수 있다. 이를 통해 광고주는 사용자에게 더욱 관련성 있는 광고를 보여주고, 광고 효과를 높일 수 있다.

### 주의

---

쿠키는 사용자의 프라이버시와 관련된 이슈를 불러일으킬 수도 있다.

사용자는 웹 브라우저 설정을 통해 쿠키의 사용을 제어하거나, 필요에 따라 특정 쿠키를 삭제할 수 있다.

# 유형

---

### 세션 쿠키 (Session Cookies)

---

웹사이트 서버가 제한된 기간 동안 일시적으로 사용하기 위해 브라우저에 전송하는 식별자(문자와 숫자의 문자열)가 포함된 파일이다.

사용자가 웹사이트를 방문하는 동안에만 브라우저에 임시로 저장된다. 사용자가 브라우저를 닫으면 세션 쿠키는 삭제된다.

이 쿠키는 사용자의 웹사이트 이동 경로를 추적하거나, 로그인 상태를 유지하는 등의 용도로 사용된다.

세션 쿠키는 기본적으로 활성화되어 있다. 세션 쿠키의 목적은 개별 웹 페이지가 더 빠르게 로드되고 웹 사이트 탐색을 개선하는 것이다.

브라우저가 이미 전송한 페이지 구성 요소를 서버에 알려주므로 서버는 이를 다시 전송하는 데 시간을 낭비하지 않는다.

세션 쿠키는 임시 쿠키라고도 한다. 임시 메모리에 저장되며 활성 브라우저 세션 중에만 사용할 수 있다. 최종 사용자는 브라우저 설정을 조정하여 세션 쿠키를 거부할 수 있지만, 이 경우 사용자 환경이 저하되는 경우가 많다.

[Session Cookie](https://www.techopedia.com/definition/4910/session-cookie)

### 영구 쿠키 (Permanent cookies)

---

사용자가 브라우저를 닫은 후에도 사용자의 기기에 남아 있는 쿠키이다. 사용자가 쿠키를 수동으로 삭제하거나 쿠키가 자체적으로 설정된 만료 날짜에 도달하여 자동으로 삭제될 때까지 기기에 남아있다.

영구 쿠키의 주요 목적은 사용자의 브라우징 세션 사이에 정보를 기억하고 유지하는 것이다. 이는 여러 브라우징 세션에 걸쳐 일관된 사용자 경험을 제공하거나 사용자의 사이트 선호 설정을 저장하는 데 유용하다.

영구 쿠키는 웹사이트가 사용자가 이전 방문에서 설정한 언어, 테마, 선호 설정 등을 기억하게 해준다.

또한, 웹사이트 방문 기록을 추적하거나 사용자를 식별하는데도 사용된다. 이 정보는 웹사이트 개선, 마케팅 연구, 개인화된 광고 제공 등 다양한 목적으로 사용될 수 있다.

그러나, 영구 쿠키는 개인정보 보호와 관련된 이슈를 일으킬 수 있다. 따라서, 사용자는 웹 브라우저의 설정을 통해 영구 쿠키를 관리하거나 삭제할 수 있다. 또한, 웹사이트는 사용자의 동의 없이 개인정보를 수집하거나 공유하는 데 영구 쿠키를 사용해서는 안된다.

### 인증 쿠키 (Authentication Cookies)

---

사용자가 웹사이트에 로그인했을 때 사용자를 식별하고, 그들의 로그인상태를 유지하는 데 사용된다. 이 쿠키를 통해 웹사이트는 사용자가 로그아웃하거나 브라우저를 닫았다가 다시 방문해도 사용자의 로그인 상태를 기억할 수 있다. 또한 사용자가 로그인한 상태에서 액세스할 수 있는 서비스나 기능에 대한 사용자의 접근을 허용한다.

### 추적 쿠키 (Tracking Cookies)

---

사용자의 인터넷 사용 행동을 추적하고 기록하는 데 사용된다. 웹사이트가 사용자의 활동을 모니터링하고, 이를 분석하여 사용자 경험을 개선하거나 개인화된 광고를 제공하는 데 도움이 된다.

추적 쿠키는 사용자가 방문한 페이지, 클릭한 링크, 사용자의 선호 언어 등 다양한 정보를 수집할 수 있다.

이러한 쿠키는 일반적으로 웹사이트가 아닌 광고 회사 등 제 3자에 의해 설정되기 때문에 **_서드파티 쿠키_**라고도 불린다.

### 좀비 쿠키 (Zombie Cookies)

---

사용자가 삭제하려고 해도 재생성되는 쿠키를 말한다. 보통 로컬 스토리지나 플래시 쿠키와 같은 브라우저의 여러 위치에 데이터를 저장함으로써 이를 수행한다.

사용자가 쿠키를 삭제하면, 이러한 다른 위치의 데이터를 사용하여 쿠키가 자동으로 재생성된다.

좀비 쿠키는 일반적으로 사용자의 웹 사용 행동을 추적하는 목적으로 사용되지만, 사용자의 개인정보 보호에대한 우려로 인해 많은 논란을 불러일으킨 쿠키 유형 중 하나이다.
