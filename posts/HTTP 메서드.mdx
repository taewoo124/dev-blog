---
title: HTTP 메서드
description: HTTP 메서드에 대해 표로 설명
tags: ["HTTP", "Network"]
---

| HTTP 요청 메서드                                      | 설명                                                           | 안전 | 멱등 | 캐시 가능 | 본문                       | 사용 예시                                             |
| ----------------------------------------------------- | -------------------------------------------------------------- | ---- | ---- | --------- | -------------------------- | ----------------------------------------------------- |
| GET                                                   | 특정 리소스의 표시를 요청. 데이터를 받기만 한다.               | ⭕️  | ⭕️  | ⭕️       | ❌                         | 웹페이지를 조회할 때                                  |
| HEAD                                                  | GET 요청과 동일한 응답을 요구하지만 응답 본문을 포함하지 않음. | ⭕️  | ⭕️  | ⭕️       | ❌                         | 서버 헤더 정보 조회, 특정 리소스의 존재를 확인할 때   |
| POST                                                  | 특정 리소스에 엔티티를 제출.                                   |
| 서버의 상태 변화나 Side effect를 일으키는데 사용된다. | ❌                                                             | ❌   | ❌   | ⭕️       | 폼 데이터를 서버로 보낼 때 |
| PUT                                                   | 목적 리소스의 모든 현재 표시를 요청 payload로 변경.            | ❌   | ⭕️  | ❌        | ⭕️                        | 특정 리소스를 업데이트할 때                           |
| DELETE                                                | 특정 리소스를 삭제.                                            | ❌   | ⭕️  | ❌        | ❌                         | 특정 리소스를 삭제할 때                               |
| CONNECT                                               | 목적 리소스로 식별되는 서버로의 네트워크 터널을 생성.          | ❌   | ❌   | ❌        | ❌                         | SSL 터널링 요청에 사용.                               |
| OPTIONS                                               | 목적 리소스의 가능한 통신 방법을 조회하는데 사용.              | ⭕️  | ⭕️  | ⭕️       | ❌                         | 웹 브라우저의 CORS preflight 요청에 사용.             |
| TRACE                                                 | 목적 리소스의 경로를 따라 메시지 loop-back 테스트.             | ⭕️  | ⭕️  | ❌        | ❌                         | 서버에서 클라이언트로의 요청 경로를 확인하는 데 사용. |
| PATCH                                                 | 리소스의 부분적인 수정을 수행하는데 사용.                      | ❌   | ❌   | ❌        | ⭕️                        | 특정 리소스의 일부분만 업데이트할 때.                 |
