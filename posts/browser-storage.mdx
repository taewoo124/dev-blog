# 정의

브라우저 저장소 또는 클라이언트 측 저장소는 서버 측 저장소와 유사한 원리로 작동하지만 사용 사례는 다르다.

사용자 컴퓨터에 데이터를 저장한 다음 필요할때 검색할 수 있는 `Javascript API` 로 구성된다.

---

# 종류

## Local Storage

---

웹 브라우저에서 키/값 쌍의 형태로 클라이언트 컴퓨터에 데이터를 저장하는 웹 스토리지 방식.

사용자가 브라우저에서 수동으로 삭제하지 않는 한 데이터는 평생 로컬 스토리지에 저장된다. 사용자가 창이나 탭을 닫아도 만료되지 않는다.

또한 브라우저의 메모리가 지워지지 않는 한 데이터는 브라우저에 남아있다.

### 특징

---

1. 저장된 데이터에 만료일이 없다.
2. 저장 용량 제한은 약 10MB.
3. 데이터가 서버로 전송되지 않는다.
4. 데이터는 일반 텍스트이므로 설계상 안전하지 않다.
5. 데이터 유형이 문자열로 제한되어 있으므로 직렬화해야 한다.
6. 서버 측이 아닌 클라이언트 측에서만 데이터를 읽을 수 있다.

## Session Storage

---

웹 브라우저에서 키/값 쌍의 형태로 클라이언트 컴퓨터에 데이터를 일시적으로 저장하는 웹 스토리지 방식.

브라우저의 세션 동안 데이터가 저장되며, 사용자가 해당 브라우저 탭이나 창을 닫으면 데이터는 자동으로 삭제된다.

다른 탭이나 창에서는 동일한 웹사이트에 접근해도 저장된 데이터에 접근할 수 없다.

### 특징

---

1. 데이터는 브라우저 세션 동안만 저장된다.
2. 저장 용량 제한은 약 5MB ~ 10MB (브라우저마다 약간의 차이가 있을 수 있다.)
3. 데이터가 서버로 전송되지 않는다.
4. 데이터는 일반 텍스트이므로 설계상 안전하지 않다.
5. 데이터 유형이 문자열로 제한되어 있으므로 직렬화해야 한다.
6. 서버 측이 아닌 클라이언트 측에서만 데이터를 읽을 수 있다.
7. 같은 도메인 내의 다른 탭이나 창에서는 독립적인 스토리지 영역을 가진다.

**_브라우저 세션_**

---

브라우저를 실행하여 탭이나 창을 열 때부터 해당 탭이나 창을 닫을 때 까지의 시간 동안 발생하는 모든 활동을 포함한 기간.

## [Cookie](http://www.theok.dev/posts/cookie)

---

웹 브라우저에서 서버와 클라이언트 간의 상태를 유지하기 위해 사용되는 작은 데이터 조각. 주로 사용자의 선호나 로그인 정보와 같은 데이터를 저장하는데 사용된다.

### 특징

---

1. 사용자의 상태 정보를 유지하는데 사용.
2. 크기는 보통 4KB로 제한된다.
3. HTTP 헤더를 통해 서버와 클라이언트 간 교환된다.
4. 만료일을 설정할 수 있으며, 만료일이 지나면 자동으로 삭제된다.
5. 특정 도메인 및 경로에 대한 스코프를 가지며, 해당 스코프 내에서만 전송된다.
6. 쿠키 데이터는 일반 텍스트이기 때문에 설계상 안전하지 않다.
7. 쿠키의 도메인과 경로 설정을 통해 언데, 어디서 쿠기가 전송될지 제어할 수 있다.
